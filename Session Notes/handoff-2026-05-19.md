## DNS auth — outbound email is now fully authenticated

**Pre-state:** theportalocal.com had only the google-site-verification TXT. No SPF, no DMARC, only a pre-existing Resend DKIM. Outbound mail (booking confirmations, vendor blasts, locals signups) was going unauthenticated → receiving servers were spam-classifying some.

**Now live on Vercel DNS:**

| Record | Value |
|---|---|
| `TXT @` (SPF) | `v=spf1 include:_spf.google.com include:_spf.resend.com ~all` |
| `TXT _dmarc` | `v=DMARC1; p=none; rua=mailto:hello@theportalocal.com; aspf=r; adkim=r` |
| `TXT resend._domainkey` | RSA public key (pre-existing, confirmed valid) |
| `TXT google._domainkey` | RSA-2048 (newly added; Workspace activation flipped to "Stop authentication" at session close) |

**Monitoring period:** DMARC `p=none` means receiving servers report alignment failures to hello@ but don't block any mail. Aggregate reports start landing in 24–48 hours. After ~2 weeks of clean reports, tighten to `p=quarantine` then `p=reject`.
