## Cross-project lessons captured (cross-tenant patterns)

1. **Marketing API `/insights` without `date_preset` returns empty for fresh ads.** Always pass `date_preset=today` (or appropriate window) in the cron query. Apply to any tenant integrating Marketing API insights. Filed as part of the new `feedback_meta_api_organic_deprecation.md` or extend `feedback_meta_api_error_diagnostics.md`.

2. **Boost top-up via lifetime_budget PATCH on existing adset** — MUCH cheaper than spawning a new campaign for the same post. Meta's learning carries forward. Pattern: `POST /{adset_id}` with `lifetime_budget=NEW_VALUE`. Read current first to avoid undershooting already-spent.

3. **Manual sync button as operator UX for cron-driven data** — anything driven by a cron tick (insights pulls, scrape jobs, ledger reconciliation) benefits from an operator-triggered immediate refresh. With throttle bypass option for the manual path so a click always refreshes.

4. **Photo-to-feature pipeline at 20-min cadence** — email photo to `haveebot@gmail.com` → image read → data module → page render → dynamic OG → editorial post → boost. Reusable for any community-sourced data ingestion (bird sightings tonight; sea turtles next; storm photos; Sandfest works-in-progress; etc.).

5. **State-based marketplace surface model (PAL Delivery / Call Direct / Reservations)** — when a marketplace has both formal-partner and informal-listing inventory, a state-based affordance model lets you surface EVERYTHING with appropriate CTAs per state. Universal pattern for any vertical with two-sided supply (catering, services, lodging, etc.).

6. **Conditioning-rule pricing** — set the price floor at the eventual rate from day one. No "Beta discount" that customers feel as a price jump later. Behavioral economics rule that applies any time you're testing a vertical with discounted pricing intent.

7. **Operator-confirm-before-charge for marketplace launches** — when fulfillment isn't guaranteed, accept-then-process beats pre-charge-then-refund. Cleaner fail mode (no charge if we can't deliver) + lower friction + lower chargeback risk. Pattern: customer submits → status `pending_review` → operator decides → if GO, send payment link via SMS; if NO GO, send sorry SMS.

8. **Filled-CTA-pill for primary nav** — when one vertical is THE primary CTA (e.g., a newly-launched encyclopedia), give it a filled-color treatment vs the outlined treatment of secondary verticals. Visual hierarchy in a pill cluster.

9. **iOS chrome safe-area for tall mobile drawers** — `max(5rem, env(safe-area-inset-bottom) + 4rem)` on `padding-bottom`. Browser chrome (URL bar, home indicator) eats 80-120px of `100dvh` on iOS + Android. Pattern applies to any tall scrollable mobile UI.
