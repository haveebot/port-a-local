Heye Lab should appear on customer-facing surfaces as a **quiet tech signal**, not a story. Acceptable placements:

- Header eyebrow ("CrossRef · by Heye Lab" in muted mono next to the logo)
- Footer credit ("© CrossRef — Powered by Heye Lab")
- Tagline copy in metadata / OG descriptions where natural

**Do NOT add body-copy paragraphs explaining Heye Lab**, such as:
- "CrossRef is a Heye Lab beta — one of a handful of tools built for industries whose daily work still runs on spreadsheets and tribal memory"
- "The lab hosts CrossRef, Port A Local, and a couple more in progress"
- Named ownership ("Owned and operated by Winston Caraker + Nick Merrill")
- Any section with a "Heye Lab" header that tells the umbrella's story

**Why:** Winston confirmed 2026-04-23 after the session-15 UX overhaul: "HL being a tech mystery is best for now." Nick (Heye Lab's owner) prefers the umbrella stay a quiet tech signal. Mystery + prominence-without-explanation is the brand posture, not a pitch deck paragraph.

**How to apply:**
- When building new customer-facing surfaces (CrossRef, PAL, future Heye Lab projects), include the eyebrow + footer but NO explanatory section.
- When writing metadata / tagline copy, "Powered by Heye Lab" is fine; avoid "a beta of the Heye Lab platform" style framing.
- If Winston explicitly asks for a body-copy paragraph about Heye Lab later, he'll say so — default is always to leave it out.
- Internal memory + DECISION_LOG entries can and should explain Heye Lab in detail (those aren't customer-facing).
