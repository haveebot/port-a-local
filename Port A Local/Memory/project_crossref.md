**CrossRef is a product-recall assistant for an industry that cannot hold its own catalog in working memory.** A rep, engineer, or specifier arrives with a fragment ŌĆö a partial SKU, a half-remembered code, a description of "that narrow thing we used last time" ŌĆö and CrossRef lands them on the intended product plus every cross-reference across manufacturers with confidence, reasons, and the spec sheet.

**Cross-reference is the last mile, not the whole road.** The recall step happens first, dozens of times a day, and almost never cleanly from an exact SKU.

**Why:** A 26-year electrical engineer can't reliably retrieve his own most-used product codes. He + his 4-person engineering team spend 60ŌĆō70% of their hours on photometrics and cross-referencing. That's not a productivity feature ŌĆö that's the job. Every rep agency runs some version of this play 20├Ś a day on spreadsheets and tribal memory. Nobody has it as software. Whoever owns the recall+cross-reference graph owns the specification flow.

**How to apply:** CrossRef is a **Heye Lab** beta project (alongside Port A Local). Same playbook as Sage Em (build V1 at V3 altitude), different industry slice. **The authoritative copy of this file lives in the `crossref/memory/` directory inside the repo** ŌĆö the workspace copy in `~/.claude/.../memory/` is a mirror, synced during dumptruck.

**Name history:** Originally named **Xwalk** (sessions 1ŌĆō13). Renamed to **CrossRef** on 2026-04-22 (session 14) after industry-vocabulary research ŌĆö "cross" is the verb reps/engineers actually use. The Heye Lab umbrella framing was formalized in the same pass. Sessions 1ŌĆō13 notes under `docs/session-notes/` retain the original name for provenance.

## Product framing (reframed 2026-04-20, session 8 preplan)

**Originally framed as:** cross-reference lookup. Input: competitor SKU. Output: equivalents.

**Reframed as:** product-recall assistant. Input: fragment / description / memory. Output: intended product ŌåÆ equivalents ŌåÆ photometric-aware substitution guidance.

**Validation event (2026-04-20):** Winston asked a 26-year EE for one of his most-crossed product codes. The engineer gave `GLN` ŌĆö which *is* a real product (H.E. Williams narrow industrial LED, indoor high-bay aisle) but was *not* the product he was trying to recall. He had partial memory. CrossRef's fuzzy-search surfaced adjacent SKUs, none matched his intent. **A veteran engineer could not hold his own daily product code in working memory.** That's not a user defect ŌĆö that's the industry's defining condition.

**Same engineer**: he + 4-person engineering team = 60ŌĆō70% of hours on photometrics + cross-referencing. Reclaiming even a fraction is headcount leverage. One engineer freed per team is the pricing anchor.

**Implications locked in:**
- Market size ~50├Ś "formal cross-ref events" ŌĆö every spec call, RFI, submittal review, bid lookup is a recall problem.
- The data flywheel runs on `typed_fragment ŌåÆ intended_product` pairs, not just `sku ŌåÆ equivalents`.
- Competitive moat widens: manufacturer cross-ref PDFs can't help someone who doesn't know what they're crossing *to*.
- Graph integrity is the asset. Confident wrong answers are worse than no answer ŌĆö a rep who specs an outdoor area light into an indoor aisle job loses the trust, not just the job.

**Scrape shape variants:**

- **no-og-flat-HTML pattern** (Crucial) ŌĆö site-wide boilerplate `<title>`, no og tags; display name comes from the navigation anchor for the current slug, description from the first non-boilerplate `<h1>`, spec PDF picked by `<slug>_DS.pdf` convention. Locale-variant sitemap filter early (`/la-es/|/eu-en/|/ca-fr/|/as-en/` etc. ŌĆö led.com sitemap had 25% non-EN-US URLs). One script splits output across multiple per-brand raw JSON files. ## Architecture decisions (locked)

- **Graph-native from commit one.** Product nodes, CrossRef edges carrying confidence / source / verification / spec deltas. Not a flat lookup table.
- **Multi-tenant scaffolding from day one.** `lib/tenants.ts` with 2M as tenant #1, schema ready for tenant #2 before tenant #2 exists.
- **Surface vs. substrate.** 2M sees "internal rep tool." We're building the industry's cross-reference infrastructure. Flipping external is a 12-second decision later.
- **Free + public surface.** `/xref` stays un-gated. Tenant scoping is a highlight lens, never a wall.
- **Stack matches Sage.** Next.js 16.2.4 + React 19.2.4 + Tailwind 4 + TypeScript, zero learning curve.
- **No milestone gating.** Winston's directive: *"we work until the next thing is done and we do the next thing."*
- **Bulletproof living-and-breathing.** Dumptruck protocol after every substantive session.

### Session 8 plan (locked 2026-04-20, refined same day)

Full phased plan with code-level detail in [[docs/session-8-preplan]]. Summary:

**Design principles (locked, every phase):** always useful ┬Ę graceful degradation ┬Ę empathetic copy ┬Ę explainability on every result ┬Ę confidence always visible ┬Ę fragment-as-default ┬Ę use every signal the query gives us.

**Phases:**
- **Phase 0** ŌĆö Telemetry foundation. Anonymous query + click logging. Without it the data flywheel is blind.
- **Phase 1** ŌĆö Honest search. Query parser, tiered `rankedSearch` (exact ŌåÆ prefix ŌåÆ trigram ŌåÆ description ŌåÆ fuzzy), "You might mean..." panel, graceful zero-result state with three forward paths (browse by manufacturer / application / describe).
- **Phase 1b** ŌĆö Description-based discovery, elevated from deferred. Engineer was reaching for a description as much as a code.
- **Phase 2** ŌĆö Honest crossrefs. Hard gates on environment + mounting + voltage. `CrossRef.reasons` array surfaced as chips.
- **Phase 3** ŌĆö Regression canary. 10+ fixture cases, JSON-extensible. Build-gated.
- **Phase 3b** ŌĆö Dev diagnostic mode. `?debug=1` exposes scoring internals.
- **Phase 4** ŌĆö H.E. Williams ingestion (the brand that exposed the gap, added under new regime).
- **Phase 5** ŌĆö Photometric regex parsing (lumens, watts, CCT, beam distribution). First bite of the 60ŌĆō70% time sink.
- **Phase 5b** ŌĆö Session-scoped project context. Soft bias on recent products, no auth, downpayment on project-aware recall.

**Non-functional:** perf budget <200ms p95, WCAG 2.1 AA, mobile-first (320pt), anonymous telemetry with 90-day retention, i18n scaffolding via `lib/copy.ts`.

**TAM anchor:** 60ŌĆō70% of a 5-engineer team's hours on photometrics + cross-referencing = pricing story is "one engineer freed per team," not "productivity feature."
